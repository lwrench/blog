## 词法作用域与动态作用域
- 词法作用域：函数的作用域在定义时就已经确定
- 动态作用域：函数的作用域是在函数调用时才决定的
```js
var value = 1;

function foo() {
    console.log(value);
}

function bar() {
    var value = 2;
    foo();
}

bar();

// 结果是 1
```
## 执行上下文栈
JavaScript引擎执行代码时不是一行一行执行的，而是一段一段执行的，存在变量提升与函数提升的问题

在执行代码时，首先遇到的就是全局代码，所以会先把全局执行上下文压入栈中，当执行一个函数的时候会将函数压入执行上下文栈
```js
function fun3() {
    console.log('fun3')
}

function fun2() {
    fun3();
}

function fun1() {
    fun2();
}

fun1();
```
上下文栈模拟情况
```js
// fun1()
ECStack.push(<fun1> functionContext);

// fun1中竟然调用了fun2，还要创建fun2的执行上下文
ECStack.push(<fun2> functionContext);

// 擦，fun2还调用了fun3！
ECStack.push(<fun3> functionContext);

// fun3执行完毕
ECStack.pop();

// fun2执行完毕
ECStack.pop();

// fun1执行完毕
ECStack.pop();

```

## 变量对象
对于每个执行上下文，都有三个重要属性：

- 变量对象(Variable object，VO)
- 作用域链(Scope chain)
- this

变量对象是与执行上下文相关的数据作用域，存储了在上下文中定义的变量和函数声明。

- 全局上下文的变量对象初始化是全局对象
- 函数上下文的变量对象初始化只包括 Arguments 对象
- 在进入执行上下文时会给变量对象添加形参、函数声明、变量声明等初始的属性值(在进入执行上下文时，首先会处理函数声明，其次会处理变量声明，如果变量名称跟已经声明的形式参数或函数相同，则变量声明不会干扰已经存在的这类属性)
- 在代码执行阶段，会再次修改变量对象的属性值

```js
function a() {
    var foo = 1;
    console.log(foo);
    function foo() {
        console.log("foo");
    }
}

function b() {
    console.log(foo);
    function foo() {
        console.log("foo");
    }
    var foo = 1;
}

a() //1
b() //[Function: foo]
```
未进入执行阶段之前，变量对象(VO)中的属性都不能访问！但是进入执行阶段之后，变量对象(VO)转变为了活动对象(AO)，里面的属性都能被访问了，然后开始进行执行阶段的操作。

它们其实都是同一个对象，只是处于执行上下文的不同生命周期。

## 作用域链
我们已经知道，js函数的作用域是词法作用域，在函数创建阶段就已经确定了作用域，这是因为每个函数内部都有一个[[scope]]属性，在函数创建阶段就会将其所有父变量对象保存到其中
```js
function foo(){
    function bar(){

    }
}
```
相当于
```js
foo.[[scope]]={
    globalContext.VO,
}
bar.[[scope]]={
    fooContext.AO,
    barContext.VO
}
```
当函数激活时，进入函数上下文，创建活动对象后，就会将活动对象加入到作用域链的前端
```js
Scope=AO.concat([[scope]])
```

以下面的这段代码为例：
```js
var scope = "global scope";
function checkscope(){
    var scope2 = 'local scope';
    return scope2;
}
checkscope();
```
1.checkscope 函数被创建，保存作用域链到 内部属性[[scope]]
```js
checkscope.[[scope]] = [
    globalContext.VO
];
```
2.执行 checkscope 函数，创建 checkscope 函数执行上下文，checkscope 函数执行上下文被压入执行上下文栈
```js
ECStack = [
    checkscopeContext,
    globalContext
];
```
3.checkscope 函数并不立刻执行，开始做准备工作，第一步：复制函数[[scope]]属性创建作用域链
```js
checkscopeContext = {
    Scope: checkscope.[[scope]],
}
```
4.第二步：用 arguments 创建活动对象，随后初始化活动对象，加入形参、函数声明、变量声明
```js
checkscopeContext = {
    AO: {
        arguments: {
            length: 0
        },
        scope2: undefined
    }，
    Scope: checkscope.[[scope]],
}
```
5.第三步：将活动对象压入 checkscope 作用域链顶端
```js
checkscopeContext = {
    AO: {
        arguments: {
            length: 0
        },
        scope2: undefined
    },
    Scope: [AO, [[Scope]]]
}
```
6.准备工作做完，开始执行函数，随着函数的执行，修改 AO 的属性值
```js
checkscopeContext = {
    AO: {
        arguments: {
            length: 0
        },
        scope2: 'local scope'
    },
    Scope: [AO, [[Scope]]]
}
```
7.查找到 scope2 的值，返回后函数执行完毕，函数上下文从执行上下文栈中弹出
```js
ECStack = [
    globalContext
];
```

## 闭包
还是一个例子
```js
var scope = "global scope";
function checkscope(){
    var scope = "local scope";
    function f(){
        return scope;
    }
    return f;
}

var foo = checkscope();
foo();
```
执行的过程是：
- 创建全局执行上下文，将全局作用域压入上下文栈
- 全局执行上下文初始化
- 执行checkscope函数，创建checkscope执行上下文，将checkscope执行上下文压入执行上下文栈
- checkscope执行上下文初始化，创建变量对象、作用域链和this
- checkscope执行结束，将checkscope上下文从执行上下文栈中压出
- 执行f函数，创建f函数执行上下文并压入栈中
- f函数执行上下文初始化，创建变量对象、作用域链和this
- f函数执行结束，将f函数的执行上下文压出栈

在f函数执行的时候，checkscope执行上下文已不在栈中，为什么f函数仍然可以访问到scope呢，这是因为f函数在f函数的在创建的时候，将checkscope函数的变量对象储存在了[[scope]]属性中，并加入到了作用域链中，所以即使checkscope已经从执行上下文栈中弹出，f函数仍然可以通过作用域链来访问checkscope的变量对象

```js
fContext = {
    Scope: [AO, checkscopeContext.AO, globalContext.VO],
}
```

现在我们来深入解析一下经典的闭包问题
```js
var data = [];

for (var i = 0; i < 3; i++) {
  data[i] = function () {
    console.log(i);
  };
}

data[0]();
data[1]();
data[2]();
```
在运行到data[0]这个函数之前，globalContext的变量对象为
```js
globalContext={
    OV:{
        data:[],
        i:3
    }
}
```
执行data[0]时，先创建执行上下文压入执行上下文栈中，然后对上下文栈进行初始化，创建变量对象，并将变量对象置于函数创建时就生成的[[scope]]属性(是一个保存有父变量对象的数组)的前列，这个新的数据就是作用域链，此时这个作用域链为
```js
data[0]Context = {
    AO:{}
    Scope: [AO, globalContext.VO]
}
```
我们已经知道AO中并没有i的值，所有js引擎会根据作用域链来寻找i的值，恰好globalContext.VO中i的值为3，即输出3

而当我们使用闭包后
```js
var data = [];

for (var i = 0; i < 3; i++) {
  data[i] = (function (i) {
        return function(){
            console.log(i);
        }
  })(i);
}

data[0]();
data[1]();
data[2]();
```
在执行data[0]这个函数前，全局执行上下文的变量对象不变
```js
globalContext={
    VO:{
        data:[],
        i=3
    }
}
```
但是初始化data[0]的执行上下文时，作用域链发生了变化
```js
data[0]Context={
    AO:{},
    Scope:[AO,匿名函数Context.AO,globalContext.VO]
}
```
在匿名函数的变量对象中存有i=0的值，所以输出0
```js
匿名函数Context = {
    AO: {
        arguments: {
            0: 0,
            length: 1
        },
        i: 0
    }
}
```
