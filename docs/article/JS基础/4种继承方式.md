## 构造函数继承
```js
function Parent(){
    this.name="parent"
}

function Child(){
    Parent.call(this)
    this.type="child"
}
```
缺点： Parent 原型链上的属性和方法并不会被子类继承

## 原型链继承
```js
function Parent(){
    this.name="parent"
}

function Child(){
    this.type="child"
}

Child.prototype=new Parent()
Child.prototype.constructor=Child
```
缺点：因为子类只进行一次原型更改，所以子类的所有实例保存的是同一个父类的值。 当子类对象上进行值修改时，如果是修改的原始类型的值，那么会在实例上新建这样一个值； 但如果是引用类型的话，他就会去修改子类上唯一一个父类实例里面的这个引用类型，这会影响所有子类实例

## 组合继承
```js
function Parent(){
    this.name="parent"
}

function Child(){
    Parent.apply(this)
    this.type="child"
}

Child.prototype=new Parent()
Child.prototype.constructor=Child
```
缺点：需要调用两次超类的构造函数

## 寄生组合继承
可以改进一下：
```js
function Parent(){
    this.name="parent"
}

function Child(){
    Parent.apply(this)
    this.type="child"
}

Child.prototype=Object.create(Parent.prototype)
Child.prototype.constructor=Child
```
