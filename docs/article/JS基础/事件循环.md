## 事件循环
浏览器中的JavaScript的执行流程和Node.js中的执行流程都是基于事件循环

JavaScript引擎仅在有任务(外部脚本加载完成后需要执行、dom回调事件、定时器任务回调等等)的时候执行，其他时间都处于等待状态，几乎不消耗CPU资源。

多个任务组成了宏任务队列
<img :src="$withBase('/宏任务.png')" alt="宏任务.png" style="display:block;margin:0 auto">

两个细节：
- 引擎执行任务时永远不会进行渲染（render）。如果任务执行需要很长一段时间也没关系。仅在任务完成后才会绘制对 DOM 的更改。
- 如果一项任务执行花费的时间过长，浏览器将无法执行其他任务，无法处理用户事件，因此，在一定时间后浏览器会在整个页面抛出一个如“页面未响应”之类的警报，建议你终止这个任务。这种情况常发生在有大量复杂的计算或导致死循环的程序错误时。

### 拆分CPU过载任务
```js
let i = 0;

let start = Date.now();

function count() {

  // 做繁重的任务的一部分 (*)
  do {
    i++;
  } while (i % 1e6 != 0);

  if (i == 1e9) {
    alert("Done in " + (Date.now() - start) + 'ms');
  } else {
    setTimeout(count); // 安排（schedule）新的调用 (**)
  }

}

count();
```

### 进度指示
通常，浏览器会在当前执行的代码完成后进行渲染（render）。任务的执行是否会花费很长时间对此没有影响。对 DOM 的更改只有在任务完成后才会被绘制。所以我们想看到代码执行的中间态，可以使用setTimeout来将任务拆分。
```html
<div id="progress"></div>

<script>
  let i = 0;

  function count() {

    // 做繁重的任务的一部分 (*)
    do {
      i++;
      progress.innerHTML = i;
    } while (i % 1e3 != 0);

    if (i < 1e7) {
      setTimeout(count);
    }

  }

  count();
</script>
```

## 宏任务和微任务
微任务仅来自于我们自己编写的代码，通常是由promise创建，还有一种`queueMicrotask(func)`，它对func进行排队，以在微任务队列中执行。

**每个宏任务之后，引擎会立即执行微任务队列中的所有任务，然后再执行其他的宏任务，或渲染，或进行其他任何操作。**

<img :src="$withBase('/任务队列.png')" alt="任务队列.png" style="display:block;margin:0 auto">

来个例子
```js
//请写出输出内容
async function async1() {
  console.log("async1 start");
  await async2();
  console.log("async1 end");
}
async function async2() {
  console.log("async2");
}

console.log("script start"); // 1

setTimeout(function () {
  console.log("setTimeout");
}, 0);

async1();

new Promise(function (resolve) {
  console.log("promise1");
  resolve();
}).then(function () {
  console.log("promise2");
});
console.log("script end");

/*
script start
async1 start
async2
promise1
script end
async1 end
promise2
setTimeout
*/
```

注意：这里的
```js
async function async1() {
  console.log("async1 start");
  await async2();
  console.log("async1 end");
}
```
相当于
```js
async function async1() {
	console.log('async1 start');
	Promise.resolve(async2()).then(() ={
                console.log('async1 end');
        })
}
```
await后面的代码会被当做微任务，整个加入到微队列中
