## 浏览器事件
- 鼠标事件
    - click
    - contextmenu
    - mouseover/mouseout
    - mousedown/mouseup
    - mousemove
- 键盘事件
    - keydown
    - keyup
- 表单事件
    - submit
    - focus
- Document事件
    - DOMContentLoaded 当html的加载和处理均完成，DOM被完全构建时
- CSS事件
    - transitionend 当css动画完成时

绑定事件可以有三种方式：
- HTML属性
```js
<input value="Click me" onclick="alert('Click!')" type="button">
```
- DOM属性
```html
<input id="elem" type="button" value="Click me">
<script>
  elem.onclick = function() {
    alert('Thank you');
  };
</script>
```
移除事件时
```js
elem.onclick = null
```
- addEventListener
```js
/**
@param event:事件名
@param hander：处理程序
@param options：
            once    为true则会在触发后自动删除监听
            capture 事件处理阶段
            passive 为true不会调用preventDefault()
*/
element.addEventListener(event, handler[, options]);
```
移除监听
```js
element.removeEventListener(event, handler[, options]);
```

## 冒泡和捕获
举个例子
```js
<div onclick="alert('The handler!')">
  <em>If you click on <code>EM</code>, the handler on <code>DIV</code> runs.</em>
</div>
```
我们是将事件绑定到div上，但是点击div的任何子标签都能触发事件，这是为什么呢？
### 冒泡
当一个事件发生在一个元素上，它会首先运行在该元素上的处理程序，然后运行其父元素上的处理程序，然后一直向上到其他祖先上的处理程序。（几乎所有的事件都会冒泡，focus事件是个例外）

event.target与this(=event.currentTarget)
- event.target表示事件触发的目标元素，在事件冒泡阶段并不会改变
- this表示事件冒泡到达的位置

### 停止冒泡
event.stopPropagation()

通常情况下我们不会停止事件的冒泡

### 捕获
DOM 事件标准描述了事件传播的 3 个阶段：

- 捕获阶段（Capturing phase）—— 事件（从 Window）向下走近元素。
- 目标阶段（Target phase）—— 事件到达目标元素。
- 冒泡阶段（Bubbling phase）—— 事件从元素上开始冒泡。

通过设置capture为true可以在捕获阶段触发事件
```js
element.addEventListener(event, handler, true);
```

## 事件委托
如果我们有许多以类似方式处理的元素，那么就不必为每个元素分配一个处理程序 —— 而是将单个处理程序放在它们的共同祖先上。

在处理程序中，我们获得 event.target，查看事件实际发生的位置并进行处理。

### action in markup
- 计数器
```html
Counter: <input type="button" value="1" data-counter>
One more counter: <input type="button" value="2" data-counter>

<script>
  document.addEventListener('click', function(event) {

    if (event.target.dataset.counter != undefined) { // 如果这个特性存在...
      event.target.value++;
    }

  });
</script>
``` 
- 切换器
```html
<button data-toggle-id="subscribe-mail">
  Show the subscription form
</button>

<form id="subscribe-mail" hidden>
  Your mail: <input type="email">
</form>

<script>
  document.addEventListener('click', function(event) {
    let id = event.target.dataset.toggleId;
    if (!id) return;

    let elem = document.getElementById(id);

    elem.hidden = !elem.hidden;
  });
</script>
```

## 浏览器默认行为
### 阻止浏览器默认行为
```html
<a href="/" onclick="return false">Click here</a>
or
<a href="/" onclick="event.preventDefault()">here</a>
```

addEventListener 的 passive: true 选项告诉浏览器该行为不会被阻止。这对于某些移动端的事件（像 touchstart 和 touchmove）很有用，用以告诉浏览器在滚动之前不应等待所有处理程序完成。

## 创建自定义事件
创建
```js
/**
    @prama type:事件类型
    @prama options:
                bubbles:true/false      事件冒泡
                cancelable:true/false   是否阻止默认行为
*/
let event = new Event(type[, options]);
```
触发 `dispatchEvent`
```html
<h1 id="elem">Hello from the script!</h1>

<script>
  // 在 document 上捕获...
  document.addEventListener("hello", function(event) { // (1)
    alert("Hello from " + event.target.tagName); // Hello from H1
  });

  // ...在 elem 上 dispatch！
  let event = new Event("hello", {bubbles: true}); // (2)
  elem.dispatchEvent(event);

  // 在 document 上的处理程序将被激活，并显示消息。

</script>
```
