## JS类型分类
- 数值(number)
- 字符串(string)
- 布尔值(boolean)
- undefined
- null
- 对象(object)
- Symble

### `typeof`关键字
```js
typeof 123 //"number"
typeof '123' //"string"
typeof false //"boolean"
typeof ()=>{} //"function"
typeof {} //"object"
typeof [] //"object"
typeof undefined //"undefined"
typeof null //"object"
```
在这里大家可能会有疑问了，为什么`typeof null`是`object`呢？

其实这是初代js设计时的一个遗留问题，typeof是根据输几类型的字节码来判断数据到底是什么类型，而null当时是被当作特殊的object来处理的，并没有独立成为一个数据类型，直到后来null独立成为新的数据类型时，为了兼容之前的代码，就一直沿用了下来。

### `instanceof`
instanceof 用来比较一个对象是否为某一个构造函数的实例

- function Object()由function Function()创建
- function Function()由function Function()创建

<img :src="$withBase('/原型链.png')" alt="原型链.png" style="display:block;margin:0 auto">

自己实现instanceof
```js
function instance_of(L,R){
    let R_prototype=R.prototype
    while(true){
        if(L.__proto__===null){
            return false
        }
        if(L.__proto__===R_prototype){
            return true
        }
        L=L.__proto__
    }
}
```

### `null`和`undefined`
null和undefined都表示没有，含义非常相似
甚至`undefined == null`**(null只与undefined `==`)**,在if语句中它们都会被转成false

在js设计之初，null用来表示空的对象，undefined用来表示此处无定义原始值
```js
Number(null) //0
Number(undefined) //NaN
```
常见的差异可以从代码中看出
```js
// 变量声明了，但没有赋值
var i;
i // undefined

// 调用函数时，应该提供的参数没有提供，该参数等于 undefined
function f(x) {
  return x;
}
f() // undefined

// 对象没有赋值的属性
var  o = new Object();
o.p // undefined

// 函数没有返回值时，默认返回 undefined
function f() {}
f() // undefined
```

## 数据类型的强制转换
强制转换主要使用Number() String() 和 Boolean()这三个函数
### Number()
```js
// 数值：转换后还是原来的值
Number(324) // 324

// 字符串：如果可以被解析为数值，则转换为相应的数值
Number('324') // 324

// 字符串：如果不可以被解析为数值，返回 NaN
Number('324abc') // NaN

// 空字符串转为0
Number('') // 0

// 布尔值：true 转成 1，false 转成 0
Number(true) // 1
Number(false) // 0

// undefined：转成 NaN
Number(undefined) // NaN

// null：转成0
Number(null) // 0

//参数是对象时，将返回NaN，除非是包含单个数值的数组
Number({a: 1}) // NaN
Number([1, 2, 3]) // NaN
Number([5]) // 5
```
关于对象类型：
- Number(obj)时会首先调动obj的valueOf()方法，如果返回的时原始类型的值，则直接对该值使用Number函数
- 如果valueOf方法返回的还是对象，则改为调用对象自身的toString方法，如果toString方法返回的原始类型的值，就对该值使用Number函数
- 如果toString方法人返回的对象，就报错


### String()
```js
String(123) // "123"
String('abc') // "abc"
String(true) // "true"
String(undefined) // "undefined"
String(null) // "null"

//String方法的参数如果是对象，返回一个类型字符串；如果是数组，返回该数组的字符串形式
String({a: 1}) // "[object Object]"
String([1, 2, 3]) // "1,2,3"
String([]) //''
```

关于对象类型，与Number相反：
- 先调用对象自身的toString方法。如果返回原始类型的值，则对该值使用String函数，不再进行以下步骤。
- 如果toString方法返回的是对象，再调用原对象的valueOf方法。如果valueOf方法返回原始类型的值，则对该值使用String函数，不再进行以下步骤。
- 如果valueOf方法返回的是对象，就报错。
### Boolean()
只有一下5个值转换结果为false：
- null
- undefined
- 0
- NaN
- ''
当然，false的值仍为false

## 数据类型的自动转换
预期什么类型的值，就调用该类型的转换函数。
- 不同类型的数据互相运算（除了'+'可能把运算子转换为字符串，其他的运算符都会自动把算子转换为数值类型）
- 对非布尔类型的数据求布尔值
- 对非数值型类型的值使用一元运算符
